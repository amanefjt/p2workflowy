# 学術翻訳・構造化ルール (Academic Translation & Structuring Rules)

本プロジェクトにおける、学術論文のMarkdown構造化および翻訳に関する基本ルールを定義する。

## 1. 構造復元 (Structure Restoration)
PDF由来の崩れたテキストをMarkdown化する際の原則。

- **見出しの厳格化**:
    - セクションタイトルには必ずMarkdownの見出し記号（`#`, `##`, `###`）を使用すること。
    - リスト記号（`-`, `*`）や数字（`1.`）を見出しとして使用することは厳禁。
    - 文脈から明らかに新しいセクションである場合は、明示的な見出しがなくとも `# Introduction` などを補う。

- **ノイズ除去**:
    - ページ番号、ヘッダー、フッター、参考文献（References）、謝辞（Acknowledgements）などは構造化の段階で削除する。
    - 一つの論文につき、`# Introduction` と `# Conclusion` はそれぞれ一つのみとする。

## 2. 実践的翻訳 (Practical Translation)
学術的正確性とWorkflowyへの親和性を両立する翻訳原則。

- **出力の隔離**:
    - 翻訳結果には、思考プロセス、指示文（ルール）、辞書定義などを一切含めないこと。
    - 翻訳テキストのみを `<result>` タグで囲んで出力し、システム側で抽出可能にする。

- **用語の統一**:
    - `glossary.csv` に定義された用語（例: Strathern -> ストラザーン）を必ず使用する。
    - 訳語に対して過剰な強調（太字 `**`）を行わない。

- **形式の維持**:
    - 原文の段落構造を維持する（勝手に統合・分割しない）。
    - 専門用語は、文化人類学の文脈に即した訳語を選定する。
